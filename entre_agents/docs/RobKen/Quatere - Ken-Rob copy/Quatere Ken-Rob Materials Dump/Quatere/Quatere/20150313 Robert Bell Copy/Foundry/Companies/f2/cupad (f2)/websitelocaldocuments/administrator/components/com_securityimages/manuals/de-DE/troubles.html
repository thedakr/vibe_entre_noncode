  1. Das Problem kann in der Datei imageGenerator.php sein. Verwenden Sie den JoomlaExplorer und durchsuchen Sie die Datei (components/com_securityimages/plugins/xxx/y.y/imageGenerator.php). Überprüfen Sie die Datei-Berechtigungen für die Datei img.php, die Sie verwenden möchten und setzen die Berechtigung auf 755. Oft wird 644 gesetzt, und es erlaubt den CAPTCHA nicht, aktiv zu sein. Es ist egal, dass der Ordner die Berechtigung 755 hat. Die Datei muss die Berechtigung 755 haben. Berechtigung von 644 bis 755 und SI wird arbeiten....
  2. PHP-GD darf nicht auf Ihrem Server installiert sein, sehen Sie bitte zuerst unter
    http://wiki.waltercedric.com/index.php/Com_securityimages_a_captcha_framework#Require nach,
    und überprüfen Sie Ihre Einstellungen!
  3. On some host, the  debug output by the script if DEBUG is set to yes. can generate: "Cannot modify header information - headers already send blablabla" errors in the webserver. Deactivate it.
  4. Überprüfen Sie die gesamten Einstellungen auf Ihrem Server und rufen Sie folgenes auf:
    http://yourhost/index.php?option=com_securityimages&task=displayCaptchaByPlugin&plugin=hncaptcha&version=1.0 und kontrollieren die Fehlermeldung
